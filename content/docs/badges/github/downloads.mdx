Show download counts for release assets with granular control over scope.

<BadgeSandbox

    { name: "owner", label: "GitHub username or org", placeholder: "atom", required: true },

/>

## URL formats

### All assets, all releases

<CodeLine language="bash" code="/github/downloads/{owner}/{repo}.svg" hideCopy />

Total downloads across every asset in every release.

### All assets, latest release

<CodeLine language="bash" code="/github/downloads/{owner}/{repo}/latest.svg" hideCopy />

Total downloads across all assets in the latest release only.

### All assets, specific tag

<CodeLine language="bash" code="/github/downloads/{owner}/{repo}/{tag}.svg" hideCopy />

Total downloads across all assets for a specific release tag.

### Specific asset, all releases

<CodeLine language="bash" code="/github/downloads-asset/{owner}/{repo}/{assetName}.svg" hideCopy />

Downloads of a single asset file summed across all releases.

### Specific asset, latest release

<CodeLine language="bash" code="/github/downloads-asset/{owner}/{repo}/{assetName}/latest.svg" hideCopy />

Downloads of a single asset in the latest release only.

### Specific asset, specific tag

<CodeLine language="bash" code="/github/downloads-asset/{owner}/{repo}/{assetName}/{tag}.svg" hideCopy />

Downloads of a single asset in a specific release tag.

## Legacy endpoint

The older `/github/assets-dl/{owner}/{repo}` and `/github/dt/{owner}/{repo}` endpoints still work and return downloads for the latest release (or a specific tag). The new `/github/downloads/` endpoints offer more flexibility.

## Data source

Uses the [GitHub Releases API](https://docs.github.com/en/rest/releases/releases). Uses the [token pool](/token-pool) for distributed rate limiting. Cached 1 hour. For "all releases" badges, paginates through up to 50 pages (5,000 releases).
