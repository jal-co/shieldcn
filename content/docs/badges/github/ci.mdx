Show the status of GitHub Actions workflows and commit checks.

<BadgeSandbox
  endpoint="/github/ci/:owner/:repo.svg"

    { name: "owner", label: "GitHub username or org", placeholder: "vercel", required: true },

  extraParams={[

    { name: "branch", label: "branch", placeholder: "main" },
  ]}
  defaults={{ owner: "vercel", repo: "next.js" }}

/>

## URL formats

### Workflow status

<CodeLine language="bash" code="/github/ci/{owner}/{repo}.svg" hideCopy />

<CodeLine language="bash" code="/github/ci/{owner}/{repo}.svg?workflow=ci.yml" hideCopy />

<CodeLine language="bash" code="/github/ci/{owner}/{repo}.svg?branch=main" hideCopy />

<CodeLine language="bash" code="/github/ci/{owner}/{repo}.svg?workflow=ci.yml&branch=main" hideCopy />

Shows the conclusion of the most recent workflow run. Filter by workflow file or branch using query parameters.

### Commit checks

<CodeLine language="bash" code="/github/checks/{owner}/{repo}.svg" hideCopy />

<CodeLine language="bash" code="/github/checks/{owner}/{repo}/{ref}.svg" hideCopy />

<CodeLine language="bash" code="/github/checks/{owner}/{repo}/{ref}/{check_name}.svg" hideCopy />

Shows the combined check status for a commit. Optionally specify a branch/tag ref or individual check name.

## Status values

| Status | Color | Description |
|--------|-------|-------------|
| `passing` | Green | All checks passed |
| `failing` | Red | One or more checks failed |
| `pending` | Amber | Checks are still running |
| `cancelled` | Gray | Workflow was cancelled |
| `skipped` | Gray | Workflow was skipped |

## Styling notes

CI badges automatically show a **colored status dot** based on the result. You can also use split mode for a two-tone background, or hide the dot entirely with `?statusDot=false`.

## Caching

CI badges are cached for 10 minutes to balance freshness with API rate limits.
